Local Machine Execution Guide

A step-by-step developer guide to spin up and test the AI Traffic Optimizer on your system.

Prerequisites Verification

Before launching, ensure you have the proper runtime environments installed. Check the system requirements checklist:

🐍 Python ≥ 3.10

  • OpenCV (`opencv-python`)
  • YOLOv8 (`ultralytics`)
  • ZeroMQ (`pyzmq`)
  • Pandas (`pandas`)

🟢 Node.js ≥ 18.0

  • Express & Helmet
  • Socket.io & ZeroMQ
  • jsonwebtoken & bcryptjs
  • morgan & dotenv

🚗 SUMO (Optional)

  • `SUMO_HOME` env variable set
  • SUMO GUI launcher
  • Used to validate AI signal flows in mock traffic networks

Step 1: Automated Environment Setup

We provide an automated setup verification script that checks your dependencies, creates default environmental values, pre-downloads the YOLO weights, and sets up a launcher.

powershell 
# Navigate to your workspace directory
cd "C:\Users\User\Desktop\delhi"

# Run the setup script
python demo_setup.py
YOLOv8 Nano Model Downloading

The setup script will fetch the yolov8n.pt weight file directly from Ultralytics servers so there is no start lag during your presentation.

Step 2: Run the Infrastructure

You can launch the complete pipeline automatically using our script, or run the components manually for finer control during your demonstration.

Double-click or run the compiled batch file. This will spawn 4 independent console windows configured in the correct execution order:

cmd 
start_demo.bat

Open 4 separate terminal windows, navigating to C:\Users\User\Desktop\delhi in each, and run these commands in order:

1. Terminal 1: Perception Engine 🎥

Performs YOLO detection on traffic feed. Shows annotated window.

python 
python detection.py --source traffic_video.mp4
2. Terminal 2: Optimizer Brain 🧠

Processes weights and schedules green phases via Priority Queue.

python 
python optimizer.py
3. Terminal 3: API Command Server 🌐

Starts WebSockets & override REST endpoint.

node 
npm start
4. Terminal 4: SUMO Traffic Simulator 🚗

Launches the visual road grid (falls back to mock if SUMO is absent).

python 
python sumo_config.py --gui

Troubleshooting Guide

Encountered an issue? Expand the items below for quick fixes:

"ultralytics not found — running in MOCK mode"
Run pip install ultralytics. If you hit PyTorch installation issues, verify you have the correct Python architecture, or install PyTorch explicitly from pytorch.org first.
"ZMQ: Address already in use" (Port conflicts)
Ports 5555 or 5556 are currently occupied by a hanging script. Find and terminate the process in Windows PowerShell using:
netstat -ano | findstr :5555
Then run taskkill /F /PID <PID> using the PID shown.
SUMO "FileNotFoundError" or "SUMO_HOME not set"
Run python sumo_config.py once first to generate simulation networks. If you see env errors, configure your local environment path:
$env:SUMO_HOME = "C:\Program Files\Eclipse\Sumo"
OpenCV: "Cannot open video source"
Verify traffic_video.mp4 is in the directory. If utilizing a webcam and experiencing errors, try alternate indices in detection.py (e.g. --source 1 or --source 2).
